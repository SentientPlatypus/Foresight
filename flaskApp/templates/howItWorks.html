How it works

its pretty simple

Integration.

Foresight splits its responsibilities into 2 applications: The webapp you are viewing right now, and the Foresight API.
The webapp is responsible for displaying content, while the API does most of the heavylifting.
I did this, because I needed to access data in javascript that is only available with python libraries.
So, I just had my scripts send queries to a python application: the Foresight API.

The Foresight API

As stated before, I made the foresight API do all the heavy lifting like webscraping and predictions. Its just a flask app that returns strings.
I made it open source. Check out its github if you're interested.

Webscraping

A lot of the data you see is webscraped from google finance like News articles and company descriptions. Why? because the google finance api has been discontinued.
The Foresight API became its replacement, using the Beautiful soup python library to scrape data from the google finance website. It works really well, and there has not been any issues lately.
Pretty much theres a function that scrapes data like news articles and static company information, and will return it when queried.

Predictions

The Foresight API uses an LSTM neural network to predict next months stock prices.
First, the API recieves the query, Then I get data from yfinance, Then i reshape the data so I can fit it into Tensorflow's Dense LSTM.
Then, I take the resulting dataset and send it back as a csv string, which is processed and displayed to you on the graph.
